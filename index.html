First thing's first! My paragraph on what semantic elements are:

"As the websites “An Overview of HTML5 Semantics” and “HTML5 Semantic Elements” have stated, semantic elements can be summarized as communicating it’s meaning to both the browser and developer. Key factors in why it’s important to use semantic elements over non-semantic elements are that it creates simplicity. With either div or span being replaced by the alternatives such as form, table, or img, the content of the webpage is now described more in detail. Other programmers will be able to differentiate between what is meant to be a header, footer, navigation bar, etc. Rather than using obscure tags without much information, using semantics will vastly improve information and what webpages are trying to convey – it creates specificity, which allows for more accessibility and will also save time in the long run."
